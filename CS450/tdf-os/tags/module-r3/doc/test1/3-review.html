1. Explain briefly three different measures of the quality of a command language.  


Consistency - The same terms always mean the same thing.

Naturalness - Intuitive design.  The commands make sense.

Reasonableness - Responses are helpful and polite instead of witty/rude. etc.


2. Explain briefly three features of a command language that are important primarily 

because of their usefulness in command files.  


Intelligent Terminal Handling

Immediate Commands

Help Facilities

Packaging of Commands into scripts etc.


3. State one advantage to use of a RUN command in a command language.  


Any possible action can be embedded in a program which can then be invoked at the user's convenience.


4. State one advantage to use of keyword arguments rather than positional 

arguments.  


More variations are possible and each command can then be better documented if recorded in a log file.


5. Why would it be important for a program to know whether it was invoked by a 

terminal command or a command file?  


Error correction must be possible if the program called was invoked by a command file?


6. What is the principal difference between submitting a batch job and running a 

command file?  


A batch job requires no user input what-so-ever.


7. Describe two desirable qualities of a command interface and illustrate each with a 

specific example.  


Echoing?

Break character?


8. List and describe briefly five characteristics of a command language that are 

desirable primarily to support command files.  


    • accept arguments like standard commands, to be substituted in their text like 

macro arguments. 

    • allow command files to invoke other command files in a nested structure. 

    • allow input to come from either the terminal or the command file. 

    • allow optional display of the commands executed on the terminal, and optional 

logging to a file or printer. 

    • allow comments in command files for documentation purposes. 

    • allow the user to intervene when an error is detected, and then resume the 

command file at a suitable point.
