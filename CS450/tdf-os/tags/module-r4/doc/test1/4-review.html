1. Explain in your own words the meaning of the process concept. 


A process is a task to completed, that is packaged in such a way that processes can be queued and executed in a constant stream.


2. Give examples of some resources that are (a) serially reusable, (b) reusable and 

shareable, and (c) consumable. 


a. Memory or a printer

b. Files

c. Blocks of data in a message


3. Which two resource types are handled in a fundamentally different way from all 

others? What is the major difference in handling, and why is it necessary? 

   

Serially reusable and consumable.  Mutual exclusion must exist to prevent processes from accessing resources which are in use by other processes.


4. How does a typical queue structure used within an operating system differ from 

the usual definition of a queue? 

  

Queue's in an operating system must allow for insertion at any point.

 

5. In the simplified process state diagram of Figure 4-3, explain why there is no 

transition from the blocked state to the running state. 

   

If a program is blocked, it is not fit to run.  Perhaps it requires use of resources which are currently being used by another process.  A process can run unless it is in the ready state exclusively.


6. Explain the distinctions among a thread, a lightweight process, and an ordinary 

process. 

   

Lightweight processes may be split into multiple threads which can access common data, unlike an Ordinary Process.  Lightweight processes were also introduced to serve limited system needs, such as a fork operation.  Threads can be considered subdivisions of an ordinary process.


7. Identify five explicit operations on processes that an OS might provide in 

response to system calls. 

   

Suspend

Resume

Create 

Destroy

Execute


8. Describe the steps performed by a typical OS during dynamic creation of a new 

process. 

   

    • Create or allocate a process control block; 

    • Allocate initial memory space for program and data; 

    • Identify and (usually) load the program to be run; 

    • Assign initial attributes and resource limits to the process; 

    • Allocate initial resources to the process, if any; 

    • Establish the starting state for the process, and setup or complete the PCB. 


9. Contrast the UNIX fork and exec mechanism with the more common 

create_process operation. 

  

fork essentially runs the same program as the parent process, but has a different data area.  The child process then calls exec which loads a new program, completely replacing the old one.


10.  Explain why it may not be desirable to destroy all the descendants of a process 

when the parent process is destroyed.  


The children processes may have been intended to have longer life cycles than the parent or to serve a purpose after the fact.
